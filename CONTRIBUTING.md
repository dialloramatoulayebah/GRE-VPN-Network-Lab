# Contributing to GRE-VPN-Network-Lab

Thank you for your interest in contributing to this GRE VPN Network Lab project!

## How to Contribute

### Adding Actual Files

This repository contains placeholder README files for binary content. To contribute the actual files:

1. **Network Topology Diagram** (`diagrams/network_topology.png`)
   - Create a professional network diagram showing the complete topology
   - Include all routers (R1, R2, ISP), networks, and the GRE tunnel
   - Use PNG format
   - Recommended tools: draw.io, Microsoft Visio, or Packet Tracer export
   - Replace `network_topology.png.readme` with `network_topology.png`

2. **Packet Tracer File** (`files/Pratique_GRE.pkt`)
   - Create a working Packet Tracer project implementing the lab
   - Apply configurations from the `configs/` directory
   - Test all connectivity before submitting
   - Ensure the file opens without errors
   - Replace `Pratique_GRE.pkt.readme` with `Pratique_GRE.pkt`

3. **Screenshots** (`screenshots/`)
   - Provide clear, high-resolution screenshots
   - Capture ping tests and tunnel status verification
   - PNG format preferred
   - Replace the `.readme` files with actual `.png` files

4. **Lab Report** (`report/Rapport_lab_pratique_GRE.docx`)
   - Create a comprehensive lab report
   - Include all sections mentioned in the placeholder
   - Use Microsoft Word format (.docx)
   - Replace `Rapport_lab_pratique_GRE.docx.readme` with actual document

### Improving Configurations

If you want to enhance the router configurations:

1. Fork the repository
2. Make your changes to files in `configs/` directory
3. Test the configurations in Packet Tracer
4. Submit a pull request with a clear description of improvements

### Reporting Issues

- Use the GitHub Issues tab to report problems
- Provide detailed information about the issue
- Include screenshots if applicable

### Code of Conduct

- Be respectful and constructive
- Help others learn
- Share knowledge and resources
- Test changes before submitting

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/GRE-VPN-Network-Lab.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -m "Description of changes"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Questions?

Open an issue for any questions or clarifications needed.
